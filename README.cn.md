多专家问答系统
多专家问答系统是一款基于 Python 的模块化 AI 对话框架。该框架将用户问题分发至多个领域专属的专家大语言模型（LLM）进行并行推理，完整记录对话历史，通过独立的安全模块过滤风险敏感内容，并借助专用的监督聚合器将分散的专家回复整合为完整的最终答案。系统支持对接多款主流大模型服务 API。
✨ 核心特性
🧠 多专家并行推理
将用户查询通过 ThreadPoolExecutor 并发分发至医疗、法律等专家模块，实现多源并行推理，提升回复生成效率。
🔐 独立安全核心模块
实现可配置的黑白名单关键词过滤机制；自动记录所有违反安全规则的输入 / 输出内容，供后续审核使用。
📝 持久化对话记忆
基于 JSON 的本地持久化存储对话上下文，自动保留最近 20 轮聊天记录，维持连贯的对话逻辑。
🎯 监督聚合核心
汇总所有独立专家智能体的原始回复内容，剔除冗余信息，为终端用户合成连贯、完整的最终答案。
🌍 双大模型 API 兼容
原生支持两大主流大语言模型服务提供商：智谱 AI（GLM-4.5-air）和阿里云百炼（Qwen-Turbo），并封装了统一的 API 请求适配器。

├── test-memory.py        # 记忆模块验证的独立测试脚本
├── .env.example          # 环境变量配置模板
├── requirements.txt      # 项目依赖包清单
├── safety_rules.json     # 可编辑的安全配置文件：自定义黑白名单关键词

🗺️ 未来开发路线图
语义化查询路由：替换简单的关键词匹配，采用基于嵌入（Embedding）的文本分类方式，将问题智能分配至匹配的专家智能体。
