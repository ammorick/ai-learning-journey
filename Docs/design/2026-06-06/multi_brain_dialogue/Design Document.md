## 关键补充说明

### 1. 安全大脑全覆盖
- **输入检测**：所有用户问题先经过安全大脑（关键词/语义过滤）。
- **中间检测**：每一轮多大脑对话生成的答案合并前，也要经过安全检测，防止有害内容进入下一轮。
- **输出检测**：最终答案返回用户前，再次进行安全过滤。
- 实现方式可以是统一调用 `is_safe()` 函数，或设计独立的微服务。

### 2. 线程池与并发优化
- **当前风险**：`ThreadPoolExecutor` 每次新建，高并发下可能资源耗尽。
- **改进方向**：
  - 复用全局线程池，设置合理的 `max_workers`（例如 CPU 核心数 * 2）。
  - 增加超时控制和任务队列长度限制。
  - 考虑是否引入多进程（`ProcessPoolExecutor`）处理 CPU 密集型任务，但 I/O 密集型（API 调用）多线程已够。
- **警示**：需要在代码中明确注释“改进线程池！不然容易爆炸！”

### 3. 响应时间
- 每轮对话的同步点需要记录开始/结束时间。
- 如果某轮总耗时超过阈值（例如 60 秒），可以中断循环并返回已有结果。
- 单个 API 调用也应有超时（已在 `api1.py` 中设置为 25 秒）。
- 未来可以动态调整超时，根据用户问题复杂度或当前负载。
