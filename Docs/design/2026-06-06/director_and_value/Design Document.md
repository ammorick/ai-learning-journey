## 详细说明

### 1. 用户提问联想
- **当前**：基于关键词路由（如“感冒” → 医学专家）。
- **未来方向**：
  - 范围如何确定？不再依赖静态数据或规则，而是基于语义相似度（如 embedding）或上下文动态推断。
  - 发散性是否仅限于行业内？可跨行业联想，但需控制边界（如通过阈值或人工设定）。
- **实现建议**：使用向量数据库或嵌入模型计算问题与专家领域的相似度，输出多个候选专家。

### 2. 总监大脑优化
- **当前**：简单合并（`[medical]xxx\n[law]xxx`）。
- **目标**：实现真正的“2合1”深度融合（如根据问题重点，将不同专家的观点有机重组）。
- **品牌/偏好模式**：允许用户选择回答风格（长篇详细 vs 精简短小），可通过系统 prompt 或后处理实现。
- **记忆框架引用**：多轮对话中产生的中间结论或共识，是否可以存入长期记忆（`memory.json`）并被后续对话引用？需要设计明确的数据结构。

### 3. 价值观念系统（最超前部分）
- **定义**：价值 = 对我有用、知识库之外、全新的内容。可能包含无效或矛盾信息。
- **“新”的识别**：若本地数据库没有，是否调用外部昂贵 API？MVP 阶段不实现，成品阶段可考虑。
- **边界制定**：是否需要人为划定行业规则？复合型问题可能产生新的行业“价值”，但行业本身也在变化。
- **筛选机制**：早期可人工定期筛选，中期难以持续，需设计自动价值评估与清理算法。
- **价值大脑**：价值体系形成后，是否需要专门的大脑来监控、更新价值，并主动介入其他大脑的对话？如何介入？目前无解，留作长远探索。

### 4. 安全监测
- 必须全覆盖：输入、输出、中间结果。
- 随时修正/定时扫描：可设计后台任务定期检查规则有效性。
